## ğŸ’¡ How Can I Contribute?

### Reporting Bugs

If you find a bug:
1.  **Check existing issues** to see if the bug has already been reported.
2.  If not, open a new issue using the **'Bug Report' template**.
3.  Include a clear title, detailed steps to reproduce, your OS, and project version.

### Suggesting New Features

If you have a great idea:
1.  **Open an issue** and select the **'Feature Request' template**.
2.  Describe the feature and its use case.
3.  Wait for feedback from a maintainer before spending time on implementation.

### Submitting a Pull Request (PR)

Follow these steps for code changes:
1.  **Fork** the repository and clone it locally.
2.  Create a new branch from `main`: `git checkout -b feature/your-feature-name`
3.  Make your changes.
4.  Ensure your code passes any existing tests and linter checks.
5.  Write clear, descriptive **commit messages**.
6.  Push your changes and [open a Pull Request](https://docs.github.com/en/github/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request) linking to the relevant issue.
